- **K-26 · Google girişi (dashboard)** · ✅ 05.07 · Firewall'ın genel erişime açık kalması kararının (K-24) üzerine ikinci bir koruma katmanı: tarayıcı kullanıcıları Google ile giriş yapar, sadece `ALLOWED_EMAILS`'teki hesaplar içeri girer. Client ID build-time gömülmez, frontend runtime'da `GET /auth/config`'ten okur — Client ID/e-posta listesi değişikliği frontend rebuild GEREKTİRMEZ.
  - **Alt not — yanlış kimlik bilgisi türü:** Kullanıcı ilk denemede Cloud Console'dan bir **Service Account** JSON'u (private key içeren) indirip paylaştı — bu ID-token doğrulama akışı için işe yaramaz ve private key sohbette paylaşılmış olduğu için güvenlik riskiydi; kullanıcıya o anahtarı silmesi söylendi.   - **Alt not — çıplak IP ile Google girişi imkansız:** Google `Authorized JavaScript origins` alanı geçerli bir genel TLD'yle bitmeyen origin'leri (çıplak IP) reddediyor — bu Google'ın sabit kısıtı, kod tarafından aşılamaz. - **K-28 · Jenkins ile otomatik dağıtım (CI/CD)** · ✅ 09.07 · `/opt/bot` gerçek bir git deposu yapıldı (önceden sadece SFTP arşivinden çıkarılmış dosyalardı — veri dosyaları `bot.sqlite`/`model.bin`/`reports/`/`.env.docker` korunarak taze `git clone`'a taşındı). Jenkins, Docker CLI + compose plugin eklenmiş özel bir imajla (`Docker-outside-of-Docker`: host'un `docker.sock`'ı bağlanır) container olarak çalışıyor; pipeline her tetiklendiğinde `/opt/bot`'ta `git reset --hard origin/main` → `docker compose build` → `docker compose up -d` → `docker inspect` ile backend sağlık kontrolü yapıyor. **Tetikleme yöntemi: GitHub webhook DEĞİL, dakikada bir cron polling** — kullanıcı, Jenkins'in (docker.sock erişimi olan, host'ta kök-eşdeğeri yetkiye sahip bir servis) internete hiç açılmamasını tercih etti; webhook için gereken gelen bağlantı riskini tamamen ortadan kaldırıyor, bedeli en fazla ~1 dakikalık gecikme.   - **Alt not — güvenlik sınıflandırıcısı üç kez müdahale etti (hepsi haklı):** (1) Jenkins kurulum sihirbazı tamamlanmadan `docker.sock` + internete açık port kombinasyonu reddedildi → önce sadece localhost'a bağlayıp güvenliği (gerçek admin hesabı, anonim erişim kapalı) kurulum ANINDA (headless groovy init script) sağlandıktan sonra ilerlendi. (3) Jenkins'i webhook için genel erişime açma reddedildi → kullanıcıya soruldu, cron polling seçildi, port hiç açılmadı.
